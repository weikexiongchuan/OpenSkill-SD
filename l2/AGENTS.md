# l2

场景层。

## 这一层的作用
- 接住用户的宽泛目标。
- 先判断当前是首次执行、续启动、恢复还是升级。
- 接入用户自有服务器时，先判断是新服务器还是现有服务器。
- 负责用户级能力聚合、回复模板、接入分流与场景编排。
- 对 lora-scripts 这类场景，也负责把“Web 训练能力判断”和“小白参数指导”接到合适的 `l1` / `l3`。
- 只组织下一步需要的 `l1` / `l3`，不替代它们。

## 用户级编排职责
- “你能做什么”的回答由 `l2` 统一生成，采用固定格式、动态拼接服务清单。
- 服务清单集中维护在 `l2/references/scene-registry.yaml`。
- “这是新服务器还是之前用过的那台”属于 `l2` 的接入分流，不属于 `l1` 或 `l3`。
- `l1` 负责状态定义与记录语义，`l3` 负责单点执行，不直接承担宽泛场景话术。

## 什么时候用
- 用户给的是宽泛目标，不是单条命令。
- 需要把多个能力拼成一个完整场景。
- 典型例子：部署或启动 ComfyUI、部署或启动 lora-scripts 训练界面、判断 lora-scripts Web 训练能力。

## 读取规则
- 先命中一个场景 skill，再按需跳到单个 `l1` 或 `l3`。
- 不要一次性把同层 AGENTS.md、多个 skill body 或全部 references 展开。
